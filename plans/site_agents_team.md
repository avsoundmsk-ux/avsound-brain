# План: команда AI-агентов по сайту av-sound.ru

Создан 2026-06-24. Статус: Шаг 1 готов, идём в Шаг 2.

## Цель
Сайт av-sound.ru (WordPress + WooCommerce, REG.RU, 1595 товаров) довести до: красивый каталог, хороший дизайн, SEO-продвижение, безопасность, соответствие законам РФ (152-ФЗ — согласия на обработку перс. данных), чтобы не заблокировали. Дальше — постоянная доработка силами команды агентов.

## Принципы (зафиксировано)
- **dev-агенты, не runtime.** Живут в `.claude/agents/`, работают внутри Claude Code, запускает Михаил/Claude. НЕ деплоим автономного агента на сайт — опасно для живой витрины + дорого.
- **Инструменты делают рутину, человек подтверждает.** Скрипты (woo.py/woo_batch.py) + субагенты + ручной гейт перед заливкой.
- **«Готовый агент с GitHub» = миф.** Берём с GitHub: промпты субагентов, MCP-серверы, skills. Не автономных «волшебников».
- **«Обучение» агента** = системный промпт + контекст-файлы (business/, studio/) + skills/MCP. Не ML.

## Шаги
### Шаг 1 — upload-инструмент ✅ ГОТОВО
- `tools/woo.py` — find/get/set-price/set-desc/set-attr/set-seo/set-images/photo, ретраи.
- `tools/woo_batch.py` — массовый параллельный конвейер (source → run --workers N), авто-детект светлого фона+перегенерация.
- Это и есть «агент, который сам работает с сайтом».

### Шаг 2 — фото каталога ПО БРЕНДАМ ⏳ ИДЁТ
- Не всё кучей. Бренд → прогон → смотрим результат → корректируем → следующий бренд.
- Текущий: **Pride** (manifest_pride.json собирается в фоне; затем run --workers 4 на ~49 шт).
- Бэклог: ~1587 товаров.

### Шаг 3 — дизайн сайта
- Доступ нужен выше REST API (wp-admin/SFTP).
- Возможные инструменты: skills `nextlevelbuilder`, `ui-ux-pro-max-skill` (проверить через find-skills); `framer-motion` — только если кастомный фронт, не для стандартной WP-темы (уточнить).

### Шаг 4 — безопасность + SEO + закон РФ
- SEO-агент: создать свой или взять промпт/MCP с GitHub (ahrefs MCP уже есть).
- Безопасность: плагины WP, формы, доступы, бэкапы.
- **152-ФЗ**: баннер согласия на обработку перс. данных, политика конфиденциальности, галочки в формах — чтобы не получить блокировку РКН.

Михаил даёт логин/пароль админки. Claude через Playwright-браузер ставит плагины, правит дизайн/настройки. SFTP — позже, точечно (код темы/хардненинг).
**Жёсткое правило: перед КАЖДОЙ правкой — полный бэкап (файлы+БД) + подтверждение Михаила.** Работаем по одному изменению → проверяем что сайт жив → следующее.

### Проверено вживую (playwright, 2026-06-24)
- WordPress 7.0 + WooCommerce, REG.RU.
- 152-ФЗ ПЛОХО: ссылка «Политика конфендициальность» (опечатка) ведёт на /refund_returns (страница возврата, НЕ политика ПД). Реальной политики/согласия нет → риск РКН.
- SEO-плагин не обнаружен (нет Yoast/RankMath) → мета голые.

### Порядок работ (ценность/риск)
1. **Бэкап** (файлы+БД) — до всего. REG.RU панель или плагин (UpdraftPlus).
2. **152-ФЗ** (приоритет, юр.риск): плагин cookie-согласия + нормальная страница «Политика обработки ПД» + чекбокс согласия в формах. Проверить регистрацию оператора ПД в РКН (юр.действие вне сайта; REG.RU = серверы в РФ ✓).
3. **SEO**: плагин (RankMath/Yoast) + мета/sitemap/robots. Мета товаров — через woo.py set-seo если плагин даёт REST.
4. **Дизайн**: тема/Customizer/page-builder.
5. **Безопасность**: Wordfence/аналог, 2FA, расписание бэкапов, сильные пароли.
