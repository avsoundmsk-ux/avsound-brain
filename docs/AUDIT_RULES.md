# AUDIT_RULES.md

# Система аудита всех изменений

Модуль: `audit_manager`.

Каждое изменение WooCommerce должно автоматически записываться.

Перед изменением товара сохранить его предыдущее состояние.
После изменения сохранить новое состояние.

---

# Что записывать для каждого изменения

* дата и время;
* Product ID;
* SKU;
* артикул;
* название товара;
* действие (создан / обновлён / удалён / пропущен);
* причина изменения;
* источник изменения (Excel);
* номер строки Excel;
* пользователь (AI Agent);
* версия программы.

---

# Для каждого изменённого поля

Сохранять тройку:

Поле / Старое значение / Новое значение.

Пример:

Название
Было: Apocalypse AAP-500.2D
Стало: Apocalypse AAP-500.2D Atom Plus

Цена
Было: 18990
Стало: 17990

Описание
Было: ...
Стало: ...

Характеристики
Было: ...
Стало: ...

Фотографии
Было: 3 фотографии
Стало: 3 новые фотографии

---

# Дополнительно по фотографиям

Сохранять:

* URL старых фотографий;
* URL новых фотографий;
* attachment_id старых фотографий;
* attachment_id новых фотографий.

---

# Хранение

Каталог: `/audit/`

Форматы:

* audit.json
* audit.xlsx
* audit.csv

Каждый запуск — отдельная папка с меткой времени:

`/audit/2026-06-27_07-45-12/`

Все изменения хранятся бессрочно.

По любому товару должна быть возможность посмотреть всю историю изменений.
