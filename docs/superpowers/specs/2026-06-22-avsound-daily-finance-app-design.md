# AVSound Daily Finance App — Design Spec

**Date:** 2026-06-22  
**Scope:** Phase 1 — Продажи (sales file parsing + report)

---

## Context

Гера ежедневно присылает 5 xlsx-файлов через Telegram:
- Продажи, Установка, Закупка, Зарплата, Рабочие покупки

Файлы экспортируются из приложения учёта расходов/доходов. Нужно считать прибыль за день без ручного ввода.

---

## Phase 1 Scope

Только файл **Продажи**. Остальные файлы — Phase 2.

---

## Input Format

Файл «Продажи» — xlsx, структура:
- Строка 1: заголовок `Список доходов по категории "Продажи"`
- Строка 2: колонки: Дата | Счет | Сумма | Валюта | ... | Теги | Комментарий
- Строки 3+: данные

Парсинг каждой строки:
- **Реализация** = колонка 3 (Сумма)
- **Закупка** = число в скобках из колонки 8 (Комментарий): `Товар (7519)` → `7519`
- **Название товара** = текст до скобок из колонки 8
- **Канал** = колонка 7 (Теги): `Авито` или пусто → `Прямые`
- **Дата** = колонка 1

---

## Calculations

По каждой строке:
- Маржа = Реализация − Закупка
- % маржи = Маржа / Реализация × 100

Итоги:
- Сумма реализации
- Сумма закупки (себестоимость)
- Валовая маржа = реализация − закупка
- % маржи итого

Разбивка по каналам: Авито / Прямые (отдельные суммы реализации и маржи).

---

## UI

**Страница одна:**

1. Зона drag & drop для одного xlsx файла
2. После загрузки — таблица:

| Дата | Товар | Канал | Реализация | Закупка | Маржа | % |
|------|-------|-------|-----------|---------|-------|---|

3. Итоговая строка внизу таблицы (жирная)
4. Карточки сверху: Реализация / Закупка / Маржа / % маржи
5. Разбивка по каналам (2 строки: Авито + Прямые)

---

## Tech Stack

- **Vite + React** — локально для Phase 1, Vercel для Phase 3
- **SheetJS (xlsx)** — парсинг в браузере, файл не уходит никуда
- **Tailwind CSS** — UI

## Success Criteria (Phase 1)

- Дропнул файл → таблица без ошибок
- Маржа считается верно (реализация − цена в скобках)
- Итоги совпадают с ручным подсчётом
- Работает на телефоне (responsive)
