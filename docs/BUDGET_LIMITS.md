# BUDGET_LIMITS.md

Budget Guard: лимиты трат и нагрузки на один запуск. Модуль обязан останавливать работу при превышении и фиксировать это в отчёте.

# Что считает Budget Guard
- стоимость генерации KIE (по числу картинок × тариф);
- количество запросов к каждому API (KIE, Brave, WooCommerce);
- суммарную стоимость текущего запуска;
- число ретраев генерации.

# Лимиты на один запуск (значения по умолчанию)
| Параметр | Значение | Примечание |
|---|---|---|
| Макс. бюджет KIE на запуск | **$5.00** (~1000 cr) | hard-stop; меняется флагом `--budget` |
| Макс. генераций на товар | **3 попытки × до 3 фото = 9** | из KIE_PROMPTS (3 ретрая) + перебор фото |
| Макс. товаров за запуск | **по умолчанию без лимита**, флаг `--limit N` | для пилотов ставить N |
| Таймаут сети (1 запрос) | **30–120 сек** | WooCommerce 120, Brave 30, скачивание 25 |
| Ретраи сети | **6** (woo) / **3** (картинки) | VPN рвёт соединение |
| Параллельных воркеров | **≤ 2** | >2 на нестабильном VPN = массовые обрывы (проверено) |
| Brave Search | бесплатно, ~2000 req/мес | следить за месячным лимитом |

# Тарифы (актуально, обновлять)
- KIE Nano Banana edit: ~**4 cr (~$0.02)** за картинку.
- Brave Search API: бесплатно (2000/мес), далее $3/мес.
- dark_bg (локальный вырез фона, без нейросети): **бесплатно**.
- tmpfiles (промежуточный хостинг фото): бесплатно.

# Поведение при превышении
1. Немедленно остановить генерацию новых картинок.
2. Завершить уже начатый товар (не бросать на полпути).
3. Записать в отчёт: «Превышен бюджет», сколько потрачено, на каком товаре остановились.
4. Уже обработанные товары НЕ откатывать.
5. State Manager помечает, докуда дошли (см. STATE_AND_RECOVERY.md) — повторный запуск продолжит.

# Предупреждения (до hard-stop)
- при достижении 80% бюджета — предупреждение в лог/отчёт.
- при balance KIE = 0 (HTTP 402) — стоп KIE, перейти на dark_bg или ручную проверку, не падать.

# Открытые параметры
- финальное значение бюджета на массовый прогон (сейчас $5);
- стоит ли разрешать fallback на бесплатный dark_bg при исчерпании KIE (по умолчанию — да, с пометкой в отчёте).
