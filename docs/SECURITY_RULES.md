# SECURITY_RULES.md

# Правила безопасности проекта

Этот документ имеет максимальный приоритет после CLAUDE.md.

Любое действие, нарушающее данные правила, считается ошибкой.

---

# Главное правило

Работать только с товарами, которые присутствуют в текущем Excel-файле.

Excel является единственным источником разрешенных товаров.

Любой товар, отсутствующий в Excel, запрещено изменять.

---

# Запрещено

Категорически запрещается:

* изменять товары, отсутствующие в Excel;
* удалять фотографии товаров вне Excel;
* менять категории товаров вне Excel;
* удалять товары;
* массово изменять каталог;
* выполнять массовую очистку медиатеки;
* выполнять массовое обновление товаров;
* автоматически исправлять существующие товары без присутствия их в Excel;
* переименовывать товары вне Excel;
* изменять URL товаров;
* изменять SKU без разрешения пользователя.

---

# Правило одной строки

Каждая строка Excel является полностью независимой задачей.

Любая операция должна иметь следующую последовательность:

Excel Row

↓

Поиск товара

↓

Product ID

↓

Изменение только этого товара

После завершения обработки переходить к следующей строке.

Никогда не выполнять операции сразу над несколькими товарами.

---

# Поиск товара

Поиск выполнять только в следующем порядке:

1. Артикул
2. SKU
3. Модель
4. Полное название
5. Похожее название

Если совпадение вызывает сомнения:

ничего не изменять.

Добавить товар в отчет:

"Требуется ручная проверка".

---

# Совпадение товара

Обновление разрешено только если подтверждено:

* совпадает артикул;
* совпадает модель;
* совпадает бренд.

Если совпадает только название —

не изменять автоматически.

---

# Работа с изображениями

Работать только с изображениями товара из текущего Excel.

Перед удалением фотографии проверить:

* принадлежит ли изображение текущему Product ID;
* используется ли изображение другими товарами;
* успешно ли загружены новые фотографии.

Если хотя бы одна проверка не пройдена —

старые фотографии не удалять.

---

# Удаление фотографий

Удалять старые фотографии разрешается только после выполнения всех условий:

✓ новые фотографии успешно найдены;

✓ успешно обработаны через KIE.ai;

✓ успешно загружены;

✓ назначены товару;

✓ проверены.

Только после этого удалять старые фотографии данного товара.

Никогда не удалять фотографии других товаров.

---

# Ошибки

Любая ошибка должна:

* записываться в лог;
* записываться в отчет;
* не останавливать обработку остальных товаров.

После ошибки переходить к следующей строке Excel.

---

# Dry Run

По умолчанию всегда использовать режим Dry Run.

Dry Run должен показать:

* какие товары будут изменены;
* какие товары будут созданы;
* какие фотографии будут удалены;
* какие фотографии будут добавлены;
* какие характеристики будут изменены.

Никаких изменений сайта выполнять нельзя.

---

# Live Run

Live Run разрешается только после подтверждения пользователя.

Без подтверждения пользователя запускать Live Run запрещено.

---

# Работа с характеристиками

Никогда не придумывать характеристики.

Если достоверная информация отсутствует —

оставить поле пустым.

Добавить товар в отчет.

---

# Работа с описанием

Не использовать ложную информацию.

Не писать характеристики, которые невозможно подтвердить.

Не рекламировать функции, отсутствующие у товара.

---

# Работа с фотографиями

Не использовать изображения:

* другого товара;
* другой модели;
* другого цвета;
* другой комплектации.

Не использовать фотографии:

* с логотипом другого магазина на товаре;
* с водяным знаком на товаре;
* с рекламными баннерами;
* с ценниками;
* низкого качества.

---

# Прерывание операции

Если хотя бы один этап вызывает сомнение —

остановить обработку именно этого товара.

Не пытаться угадать правильное решение.

---

# Приоритет безопасности

При конфликте требований всегда соблюдать следующий порядок:

1. Безопасность данных.
2. Целостность каталога.
3. Правильность информации.
4. Качество фотографий.
5. Скорость выполнения.

---

# Финальное правило

Лучше не изменить один товар, чем случайно изменить сто правильных.

Безопасность всегда важнее автоматизации.
