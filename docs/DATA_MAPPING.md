# DATA_MAPPING.md

Полное соответствие Excel → WooCommerce. Основано на реальной структуре сайта av-sound.ru (проверено на товаре id 41560 «Pride Uno plus») и реальном Excel `Копия склад (8).xlsx`.

# Как устроен товар на сайте (факт)
- **SKU** существует и генерируется сайтом по шаблону `BRAND-TYPE-NAME-NNNN-HASH` (напр. `PRIDE-MONOBLOK-PRIDE-UNO-PLUS-0071-5A7B1D`). В Excel SKU НЕТ.
- **Бренд** хранится как локальный атрибут товара **«Производитель»** (`attributes[].name="Производитель"`, `options=["Pride"]`). Поле WooCommerce `brands` НЕ используется (пустое). Таксономии бренда нет.
- **Категория** — `categories[]` по `id` (напр. 256 = «Усилители»). slug в данных мусорный, ориентируемся на id.
- **Характеристики** — локальные атрибуты (`id:0`, `visible:true`): «Тип», «Серия», «Мощность RMS…», «Размеры, мм» и т.д.
- **SEO** — плагин **SiteSEO**, мета-ключи `_siteseo_meta_title`, `_siteseo_meta_description` (дублировать в rank_math/yoast — безопасно, как запас).
- Тема **WooDmart** (куча мета `_woodmart_*`) — НЕ трогаем.

# Маппинг колонок Excel → WooCommerce

| Excel | Назначение | Обяз. | WooCommerce | Преобразование |
|---|---|---|---|---|
| A Статус | служебное (Avito) | — | — | не импортируется |
| B состояние | служебное | — | — | не импортируется |
| C GoodsType | тип Avito | — | — | вспомогательно для категории |
| D категория | категория товара | да | `categories: [{id}]` | текст → id по CATEGORY_RULES.md; при неоднозначности — ручная проверка |
| E название | имя товара | **да** | `name` | trim; бренд+модель парсятся (см. MATCHING_RULES) |
| F стоимость | цена | да | `regular_price` | число → строка, без пробелов/₽ |
| G количество | остаток | да | `stock_quantity` + `manage_stock:true` | int; 0 → `stockstatus` outofstock (уточнить политику) |
| H закуп | закупка | нет | — (НЕ публиковать) | хранить только локально/в отчёте, на сайт не писать |
| I пометка | примечание | нет | — | игнор |

# Производные поля (не из колонок, а вычисляемые)
| Поле WC | Источник | Правило |
|---|---|---|
| `attributes` «Производитель» | бренд из секции/названия | одно значение |
| `attributes` (характеристики) | поиск в сети (product_enricher) | только подтверждённые; нет данных → пусто + отчёт |
| `description` | генерация по подтверждённым данным | без выдумок |
| `short_description` | генерация | кратко |
| `_siteseo_meta_title` | генерация | из бренд+модель+категория |
| `_siteseo_meta_description` | генерация | SEO |
| `images` | поиск фото → KIE → загрузка | см. IMAGE_RULES / KIE_PROMPTS |
| `sku` | при СОЗДАНИИ — по шаблону сайта `BRAND-TYPE-NAME-NNNN-HASH` | при обновлении SKU НЕ менять |

# Правила преобразования
- Цена: `"5600"`, `5600.0` → `"5600"`. Дробные — округление по политике (TODO: до целого ₽?).
- Количество: float `2.0` → int `2`.
- Бренд: нормализовать (`MB Quart`/`Quart` → единое значение бренда — уточнить канон).
- Категория D: маппинг текст→id (CATEGORY_RULES.md). «Автоакустика» ≠ конечная категория сайта (там «Акустика»/«Сабвуферы»/…) — нужна таблица соответствия.

# Что НЕ трогаем при обновлении
- существующий корректный SKU;
- `_woodmart_*` мета (тема);
- категорию, если в Excel она не задана однозначно;
- поля, заполненные на сайте корректно и отсутствующие в Excel;
- закуп (H) — никогда не публикуется.

# Утверждённые решения
- **Канон бренда**: единое каноническое имя (MB Quart, JBL, Ural, Nakamichi…); все варианты написания приводятся к канону.
- **Stock = 0**: товар НЕ скрывать — оставить опубликованным, `stock_status = outofstock` («Нет в наличии»), карточка обновляется, история не удаляется.
- **Цена**: НЕ округлять, брать значение из Excel как есть (авто-ценообразование — отдельный модуль в будущем).
- **Категории**: НИКОГДА не создавать автоматически; нет однозначного соответствия → ручная проверка (см. CATEGORY_RULES.md).
- Категория D→id: таблица в CATEGORY_RULES.md.
