# Назначение

State Manager: устойчивость к обрывам. Повторный запуск продолжает с последнего успешно обработанного товара, не начиная Excel сначала.

# Состояние (что хранить)
- идентификатор запуска;
- путь/хэш Excel-файла;
- по каждой строке: статус (pending / done / failed / skipped / manual_review);
- Product ID результата;
- метка времени.

# Идемпотентность
- Повторный запуск того же Excel НЕ дублирует товары.
- Уже `done` строки пропускаются (если не задан --force).
- `failed` — повторять; `manual_review` — не трогать автоматически.
