# CATEGORY_RULES.md

Правила назначения категории WooCommerce товару из Excel.

# Принципы (утверждено)
- НЕ создавать новые категории автоматически.
- Категория из Excel не сопоставлена → товар НЕ создавать → ручная проверка.
- Несколько похожих категорий → ручная проверка.
- Категория назначается ТОЛЬКО по утверждённой таблице соответствия (ниже).
- При обновлении товара категорию НЕ менять, если текущая корректна.
- При создании товара категория обязательна.

| «категория» | — | — | — | артефакт строки-заголовка, игнорировать |

# Решение по confidence
- **HIGH** → назначать автоматически по таблице.
- **LOW / нет в таблице / несколько кандидатов** → НЕ назначать, товар в ручную проверку.

# Открытые вопросы (нужно утвердить)
1. Дубли имён: писать в верхний уровень (256/294/320) или в дочерний (312/310/321)? Предлагаю верхний.
2. «Автоакустика» (71 товар) — самая массовая и неоднозначная. Варианты:
   а) всё в «Акустика»(252) как родитель (MEDIUM);
   б) определять подкатегорию по типу из названия (Компонентная→295, Коаксиальная→299, Мидбас→303, Твитер/ВЧ→293, СЧ→306) — точнее, но нужен под-маппинг по словам;
   в) ручная проверка.
   Предлагаю (б) + fallback в ручную проверку.
3. «Камеры» (1 товар) — ручная проверка.
