# REPORT_FORMAT.md

# Формат отчетов

Каждый запуск программы должен создавать подробный отчет.

Отчет должен позволять понять:

* что было обработано;
* что было изменено;
* что было создано;
* какие ошибки возникли;
* что требует ручной проверки.

Отчет обязателен после каждого запуска.

---

# Общая информация

В начале отчета вывести:

Дата запуска

Время запуска

Версия программы

Режим запуска

* Dry Run
* Live Run
* Image Only
* Data Only

Имя Excel-файла

Количество строк Excel

Время выполнения

---

# Общая статистика

Вывести:

* Всего товаров
* Найдено товаров
* Создано новых товаров
* Обновлено товаров
* Пропущено товаров
* Ручная проверка
* Ошибок
* Предупреждений

---

# Статистика изображений

Вывести:

* Найдено изображений
* Загружено изображений
* Сгенерировано через KIE.ai
* Повторных генераций
* Удалено старых изображений
* Ошибок генерации
* Ошибок загрузки

---

# Информация по каждому товару

Для каждого товара вывести:

Номер строки Excel

Название товара

Артикул

Бренд

Product ID

Ссылка на товар

Статус

Возможные статусы:

* Найден
* Создан
* Обновлен
* Пропущен
* Ошибка
* Требуется ручная проверка

---

# Изменения

Показать что изменилось:

✓ Название

✓ Цена

✓ Категория

✓ Бренд

✓ Описание

✓ Краткое описание

✓ Характеристики

✓ Атрибуты

✓ SEO

✓ Главное фото

✓ Галерея

---

# Изображения

Для каждого товара вывести:

Количество найденных фото

Количество выбранных фото

Количество успешно обработанных

Количество успешно загруженных

Количество удаленных старых фото

Количество отклоненных фото

Причина отклонения (если есть)

---

# Источники данных

Показать:

Откуда были получены характеристики

Например:

* Производитель
* Официальный каталог
* Дилер
* Интернет-магазин

Также указать источник изображений.

---

# Ошибки

Если возникла ошибка, записать:

Время

Название товара

Product ID (если существует)

Этап

Описание ошибки

Рекомендация

Ошибка одного товара не должна останавливать обработку остальных.

---

# Предупреждения

Отдельный раздел:

Требуется ручная проверка.

Причины:

* Несколько совпадений
* Не найден товар
* Не найдено изображений
* Нет характеристик
* Конфликт данных
* Низкое качество фото

---

# Финальная сводка

После завершения вывести:

✓ Все товары обработаны

или

✓ Обработка завершена с предупреждениями

или

✓ Обработка завершена с ошибками

Показать:

Общее время работы

Процент успешно обработанных товаров

Количество созданных карточек

Количество обновленных карточек

Количество товаров без изменений

Количество товаров для ручной проверки

---

# Экспорт отчетов

Автоматически сохранять:

* report.csv
* report.xlsx
* report.json
* execution.log

Все отчеты сохранять в папку:

/output

Логи сохранять в:

/logs

---

# Главный принцип

После просмотра отчета пользователь должен понимать полную историю обработки каждого товара без необходимости изучать код программы.
