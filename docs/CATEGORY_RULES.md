# CATEGORY_RULES.md

Правила назначения категории WooCommerce товару из Excel.

# Принципы (утверждено)
- НЕ создавать новые категории автоматически.
- Категория из Excel не сопоставлена → товар НЕ создавать → ручная проверка.
- Несколько похожих категорий → ручная проверка.
- Категория назначается ТОЛЬКО по утверждённой таблице соответствия (ниже).
- При обновлении товара категорию НЕ менять, если текущая корректна.
- При создании товара категория обязательна.

# Решение по confidence
- **HIGH** → назначать автоматически по таблице.
- **LOW / нет в таблице / несколько кандидатов** → НЕ назначать, товар в ручную проверку.

# Утверждённые решения (верхний уровень)
1. Дубли имён → пишем в **верхний уровень**: Усилители **256**, Сабвуферы **294**, Магнитолы **320**. ✅
2. «Автоакустика» → попадает в дерево **«Акустика» (252)**, далее уточняется подкатегория (см. ниже). ✅
3. «Камеры» (1) → **ручная проверка**. ✅

# ИНТЕЛЛЕКТУАЛЬНОЕ ОПРЕДЕЛЕНИЕ ПОДКАТЕГОРИИ

Категория верхнего уровня (из Excel) — это только старт. Точная подкатегория определяется анализом:
1. название товара (колонка E);
2. характеристики (найденные product_enricher);
3. описание;
4. официальный сайт производителя;
5. категория производителя;
6. тип товара.

Нельзя определять подкатегорию только по Excel. При сомнении между несколькими подкатегориями — **НЕ угадывать**, отправить в ручную проверку.

## Принцип расширяемости
Правила (ключевые слова → подкатегория) ведутся ТОЛЬКО в этом документе и в конфиге (`config/category_rules.yml` в будущем). Код читает правила отсюда; добавление новой подкатегории/слова = правка документа+конфига, без изменения кода.

## Статусы подкатегорий на сайте
- **id есть** → назначать автоматически (HIGH).
- **нет на сайте** (категория в списке желаемых, но отсутствует в WooCommerce) → НЕ создавать категорию автоматически → товар в ручную проверку, пометка «нужна категория X».

# Алгоритм назначения (резюме)
1. Excel-категория → родитель верхнего уровня (256/294/320/252 по таблице).
2. Прогнать название+характеристики+тип через правила ключевых слов → подкатегория с id.
3. Если ровно одна подкатегория с id уверенно → назначить её.
4. Если несколько подкатегорий-кандидатов (конфликт слов) → ручная проверка.
5. Если подкатегория определена, но её НЕТ на сайте → ручная проверка (категорию не создавать).
6. Если ничего не определилось → оставить родителя верхнего уровня (fallback) ТОЛЬКО для «Акустика/Сабвуферы/Усилители/Магнитолы»; если и родитель неясен → ручная.
7. При обновлении существующего товара категорию НЕ менять, если она уже корректна.

# Конфликты (пример)
- «коаксиальная компонентная» (оба слова) → конфликт → ручная.
- «сабвуфер активный в коробе» → активный(297) vs короб(311) → ручная.
