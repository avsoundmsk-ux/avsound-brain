# AI Project Rules

# Project

Этот проект разрабатывается для автоматической обработки товаров интернет-магазина AV-Sound.ru.

Основная задача проекта:

* импортировать товары из Excel;
* находить существующие товары на сайте;
* создавать отсутствующие товары;
* автоматически заполнять описание;
* автоматически заполнять характеристики;
* автоматически искать лучшие изображения;
* обрабатывать изображения через KIE.ai;
* загружать изображения в WooCommerce;
* поддерживать единый стиль всего каталога.

Главная цель проекта — максимально автоматизировать наполнение каталога без риска повредить существующие данные.

---

# Основной принцип

Никогда не предполагай.

Если информации недостаточно — остановись и сообщи об этом.

Лучше ничего не изменить, чем изменить неправильно.

---

# Источник истины

Единственным источником данных является Excel-файл, переданный пользователем.

Разрешено работать только с товарами, которые присутствуют в текущем Excel.

Любой товар, отсутствующий в Excel, считается запрещенным для изменений.

---

# Главное правило безопасности

Запрещено:

* изменять товары вне Excel;
* удалять фотографии товаров вне Excel;
* менять категории товаров вне Excel;
* массово изменять каталог;
* массово очищать медиатеку;
* автоматически исправлять товары, которых нет в Excel;
* переименовывать товары вне Excel;
* удалять товары.

Каждая операция должна иметь следующую цепочку:

Excel Row → Product ID → Изменение.

Если нет строки Excel — изменение запрещено.

---

# Приоритеты

Приоритет выполнения задач всегда следующий:

1. Безопасность данных.
2. Правильное совпадение товара.
3. Корректность характеристик.
4. Качество изображений.
5. Производительность.

Скорость никогда не важнее безопасности.

---

# Перед началом любой работы

Перед выполнением любой задачи обязательно:

1. Прочитать CLAUDE.md.
2. Прочитать все документы из папки docs.
3. Проверить архитектуру проекта.
4. Проверить ограничения безопасности.
5. Только после этого приступать к выполнению задачи.

Никогда не игнорировать документацию проекта.

Если документация противоречит запросу пользователя — сначала сообщить о противоречии.

---

# Разработка

Никогда не писать большой монолитный код.

Разделять проект на независимые модули.

Каждый модуль должен выполнять только одну задачу.

Избегать дублирования кода.

Использовать понятные имена функций.

Добавлять комментарии только там, где они действительно нужны.

Все ошибки должны логироваться.

---

# Работа с товарами

Работать только с товарами текущего Excel.

Не выполнять никаких действий над товарами, которых нет в Excel.

При сомнительном совпадении товара ничего не менять.

Отправлять товар в отчет "Требуется ручная проверка".

---

# Работа с изображениями

Всегда стремиться использовать официальные изображения производителя.

Если официальных изображений нет — использовать только качественные фотографии.

Никогда не использовать изображения:

* с чужими логотипами на товаре;
* с чужими водяными знаками на товаре;
* с ценниками;
* с рекламными баннерами;
* с низким качеством.

После обработки через KIE.ai товар должен оставаться абсолютно идентичным оригиналу.

Разрешено менять только фон.

---

# Проверка перед изменением

Перед любым изменением обязательно проверить:

* найден ли правильный Product ID;
* совпадает ли артикул;
* совпадает ли модель;
* совпадает ли бренд;
* не используется ли изображение другим товаром.

Если хотя бы одна проверка не пройдена — ничего не менять.

---

# Режимы работы

По умолчанию всегда использовать Dry Run.

Никогда автоматически не запускать Live Run.

Live Run разрешается только после подтверждения пользователя.

---

# Отчеты

После выполнения любой операции всегда создавать подробный отчет.

Отчет должен содержать:

* выполненные действия;
* ошибки;
* предупреждения;
* пропущенные товары;
* товары для ручной проверки.

---

# Если возникла неопределенность

Не придумывать решение.

Не заполнять данные самостоятельно.

Не угадывать характеристики.

Не изменять товары.

Остановиться и сообщить пользователю, какая информация требуется.

---

# Главное правило проекта

Безопасность важнее скорости.

Правильность важнее количества.

Качество важнее автоматизации.

Лучше пропустить один товар, чем испортить весь каталог.
