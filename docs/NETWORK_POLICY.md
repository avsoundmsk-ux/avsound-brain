# NETWORK_POLICY.md

Нестабильная сеть (VPN/timeout/SSL/connection reset) — НОРМАЛЬНОЕ состояние проекта, не повод останавливать работу.

# Правила
- Любой сетевой запрос автоматически повторять.
- Экспоненциальный backoff: ожидание `BACKOFF_BASE * 2^attempt`, капы: база 3с, максимум 60с, попыток `WOO_RETRIES` (по умолч. 8).
- Прогресс сохранять после КАЖДОГО успешного шага (state_manager).
- После восстановления связи — продолжать с места (resume), не начинать заново.
- Сетевой сбой писать в audit (событие), но не считать фатальным.

# Останавливаться ТОЛЬКО если
- исчерпан лимит повторных попыток на одном запросе;
- исчерпан бюджет (budget_guard);
- логическая ошибка данных;
- требуется подтверждение пользователя (первый write, удаление, массовость).

# Реализация
- `woocommerce_client._request_with_backoff()` — общий HTTP с backoff (GET/PUT).
- Настройка через .env: `WOO_RETRIES`, `WOO_TIMEOUT`, `WOO_BACKOFF`.
- Скачивание/генерация (Brave/KIE/tmpfiles) — аналогичный backoff в своих модулях.

# Приоритет
Дойти до результата важнее, чем быстро. Долгое выполнение из-за сети допустимо.
