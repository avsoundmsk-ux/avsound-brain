# Архитектура проекта

Цель проекта — создать масштабируемую систему, которую можно развивать годами без переписывания.

Каждый модуль должен выполнять только одну задачу.

Никаких огромных файлов по несколько тысяч строк.

# Главный контроллер

Весь процесс должен запускать только один файл.

Например:

main.py

Он ничего не делает самостоятельно.

Он только вызывает остальные модули.

---

# Расширяемость

Архитектура должна позволять легко подключить:

* OpenAI;
* Gemini;
* Claude;
* Flux;
* Nano Banana;
* Seedance;
* Midjourney;
* Stable Diffusion;
* другие генераторы.

Без переписывания существующего кода.

---

# API

Каждый внешний сервис должен иметь собственный клиент.

Никаких смешанных запросов.

Например:

WooCommerceClient

KieClient

GoogleImageClient

OpenRouterClient

Каждый отвечает только за свой API.

---

# Конфигурация

Все настройки должны храниться отдельно.

Никаких API-ключей внутри кода.

Использовать .env.

---

# Логирование

Каждый модуль пишет собственный лог.

Ошибки не должны теряться.

---

# Тестируемость

Каждый модуль должен тестироваться отдельно.

Замена одного модуля не должна ломать остальные.

---

# Главный принцип

Если через год потребуется заменить KIE.ai на другую модель, достаточно заменить один модуль, а не переписывать весь проект.
